---

# 🔐 1. CORE PROBLEM

---

Currently:

* Sellers can see orders/listings of other sellers
* Approval/rejection visible to all sellers
* Data is NOT isolated

---

❌ This is a **major security issue**

---

# ✅ 2. REQUIRED SECURITY RULE

---

## 🔒 Each Seller must ONLY see:

* Their own listings
* Their own orders
* Their own customers (only those who purchased from them)

---

## ❌ Sellers MUST NOT see:

* Other sellers’ products
* Other sellers’ orders
* Other sellers’ customers

---

# 🧠 3. DATA ISOLATION LOGIC (MANDATORY)

---

## Every product must include:

```js
seller_id
```

---

## Every order must include:

```js
seller_id
```

---

## 🔐 Query Rule:

```js
FETCH data WHERE seller_id = current_logged_in_seller
```

---

## ❗ NEVER fetch global data for sellers

---

# 🏪 4. SELLER ORDER PAGE FIX

---

## ❌ Current Problem:

* All sellers see all orders

---

## ✅ Fix:

---

### Seller should ONLY see:

* Orders for THEIR products

---

## Example:

If Seller A posted Dog A:

✔ Only Seller A sees orders for Dog A
❌ Seller B should NOT see it

---

# ✔️ 5. APPROVAL / REJECTION CONTROL

---

## ❌ Current Problem:

* All sellers can approve/reject

---

## ✅ Fix:

---

### Only the product owner can:

* Accept order
* Reject order

---

## Rule:

```js
IF order.seller_id == current_user_id:
    SHOW Accept/Reject buttons
ELSE:
    HIDE buttons
```

---

# 🔐 6. CUSTOMER DATA SECURITY

---

## Seller can ONLY see:

* Buyer details related to THEIR orders

---

## ❌ Must NOT access:

* Other buyers
* Global customer data

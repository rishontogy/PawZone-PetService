PAWZONE – UNIVERSAL BACK BUTTON SYSTEM

CRITICAL ISSUE:

Many pages across Buyer, Seller, Transporter, and Admin accounts either:

❌ Have no Back button
❌ Have inconsistent Back buttons
❌ Navigate incorrectly
❌ Break page state

Implement one consistent navigation system across the entire application.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
GLOBAL BACK BUTTON RULE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every page except the main dashboard pages must have:

◀ Back

Position:
Top-left of page header

Style:
✔ Modern rounded button
✔ Same design system across app
✔ Icon + Text
✔ Mobile friendly
✔ Tablet friendly
✔ Desktop friendly

Function:

Use browser/app history navigation.

Example:

navigate(-1)

or equivalent router history back function.

Do NOT redirect to fixed pages.

This ensures:

✔ Previous page restored
✔ Filters preserved
✔ Search preserved
✔ Scroll position preserved
✔ Form state preserved

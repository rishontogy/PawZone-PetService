PAWZONE – ACCOUNT OTP VERIFICATION SYSTEM

Implement a secure OTP verification system for Buyer, Seller, and Transporter accounts.

IMPORTANT:
Do NOT change any existing signup, approval, or login workflow except where mentioned below.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
OTP GENERATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Whenever ANY new user registers (Buyer, Seller, or Transporter):

Automatically generate a unique 6-character OTP.

Rules:

• Length: Exactly 6 characters
• Uppercase letters only (A-Z)
• Numbers (0-9)
• Combination of both letters and numbers
• Example:
  A4K8P2
  8BZ4Q1
  H9X2LM

Store this OTP securely in the database.

DO NOT display this OTP anywhere inside the application.

It is for verification purposes only.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
BUYER WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Current Flow:

Buyer fills signup form
↓

Clicks Sign Up

NEW FLOW:

1. Save buyer details in the database.
2. Generate and store the OTP.
3. Open an OTP Verification popup.

Popup Title:

Verify Your Email

Message:

"Please verify your email for the OTP."

Show:

• 6-character OTP input
• Verify button
• Re-enter/Clear button

If entered OTP matches the stored OTP:

✔ Mark buyer as verified.
✔ Activate the account.
✔ Display:
  "Email verified successfully."

Then continue with the existing signup success flow.

If OTP is incorrect:

Show:

"Incorrect OTP. Please try again."

Allow unlimited attempts.

NO expiry time.

NO countdown timer.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SELLER WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Current workflow remains unchanged.

Seller:

Registers

ONLY AFTER APPROVAL:

• Verify button
• Re-enter button

If OTP matches:

✔ Seller account becomes verified.
✔ Seller can access the dashboard.

If incorrect:

Show:

"Incorrect OTP. Please try again."

Unlimited attempts.

No expiry.

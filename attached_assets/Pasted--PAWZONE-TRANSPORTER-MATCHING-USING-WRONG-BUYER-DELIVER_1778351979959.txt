# PAWZONE – TRANSPORTER MATCHING USING WRONG BUYER DELIVERY POINT FIX

Fix the CRITICAL transporter matching bug in the PawZone application.

Currently:
✅ Buyer order correctly stores the newly added delivery points for that specific order.

BUT:

❌ Transporter filtering/matching is STILL using:
buyer.savedDeliveryPoints

instead of:
order.customDeliveryPoints

This causes wrong transporters to receive requests.

---

# 1. REQUIRED FIX

---

Transporter matching MUST prioritize:

```js id="fix001"
order.customDeliveryPoints
```

ONLY.

If custom delivery points exist for that order:
DO NOT use:
❌ buyer.savedDeliveryPoints

# 4. FIX THE TRANSPORT MATCHING ENGINE

---

```js id="wrong001"

buyer.savedDeliveryPoints
buyer.addresses
buyer.profile.deliveryPoints
```

```js id="correct001"

```
